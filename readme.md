Модуль `validator` выполняет две основные задачи:
- **Дедупликация:** Проверяет `messageId`. Если сообщение уже обработано — игнорирует его.
- **Dead Letter Channel:** Сообщения, не прошедшие валидацию (например, кривая JSON-схема или пустые поля), не удаляются. `validator` формирует из них "dead letters" очередь, оборачивая их метаданными (ошибка, timestamp, исходный топик) и отправляет в топик `dead-letter-queue`.
